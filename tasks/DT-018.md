## Description

Final quality pass on the deployed production app before the MVP is considered shippable. Focus on immersion-breaking bugs, audio issues, and AI response quality.

---

## Acceptance Criteria

### Immersion Quality

- [ ] AI never breaks character or switches to English unprompted
- [ ] AI corrections are always woven naturally into responses (never blunt)
- [ ] Each persona sounds and feels distinctly different
- [ ] Ambient audio loops without audible click or gap
- [ ] TTS voice matches the persona (e.g. Helga sounds older and Bavarian)

### UX Polish

- [ ] No layout breaks on iPhone 14 / Samsung Galaxy S screen sizes (test in browser DevTools)
- [ ] No layout breaks on 1280px and 1440px desktop widths
- [ ] Loading states visible for all async operations (STT, AI response, TTS)
- [ ] Error states are friendly and in German where appropriate (e.g. "Etwas ist schiefgelaufen. Bitte versuche es nochmal.")
- [ ] No orphaned `console.log` statements in production build

### Performance

- [ ] Lighthouse score on production URL:
  - Performance: > 80
  - Accessibility: > 90
- [ ] First AI response returned within 3 seconds of transcript submission

### Known Non-Goals (do not fix in this ticket)

- Mobile app
- User accounts / auth
- Payment integration

---

## Bug Triage Process

1. Find a bug → open a GitHub Issue tagged `bug`
2. Fix in a separate branch → PR → merge to main
3. Retest on production after deploy

---

## Definition of Done

All checklist items pass on the deployed Vercel URL. Lighthouse scores meet targets. MVP is declared shippable.
