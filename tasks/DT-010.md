## Description

Teach the AI to correct the user's German mistakes organically — woven into the conversation, never breaking character or stopping the flow.

This is a prompt-engineering task, not a code-heavy task. The correction behavior lives in the system prompt and response parsing layer.

---

## Acceptance Criteria

### System Prompt Rules

- [ ] System prompt includes a correction block:

```
When the user makes a German grammar or vocabulary mistake:
- Do NOT say "That is incorrect" or break character.
- Naturally echo the correct form in your reply.
- Example: If user says "Ich habe gestern nach Berlin gefahren", 
  reply with something like "Ah, du bist also gestern nach Berlin gefahren! Wie war es?"
- Only correct 1 mistake per turn. Do not over-correct.
- After correction, continue the conversation as normal.
```

### Mistake Logging

- [ ] `POST /api/chat` response includes an optional `correction` field:

```ts
{
  reply: string;
  correction?: {
    original: string;   // what the user said wrong
    corrected: string;  // the right form
    type: string;       // "grammar" | "vocabulary" | "case"
  }
}
```

- [ ] When `correction` is present, the UI shows a subtle inline card (not blocking) below the message
- [ ] Correction is saved to `language_memory` table via DT-009

### UI Behavior

- [ ] Correction card is dismissible
- [ ] Card uses amber accent color, not red (encouraging, not punishing)
- [ ] Card shows: "💡 Du meinst: **[corrected]**"

---

## Definition of Done

Make a deliberate grammar mistake. AI naturally incorporates the correct form in its reply. A subtle correction card appears below the user's message.
