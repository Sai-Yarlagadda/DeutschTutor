## Description

Build the `/scenarios` page where the user picks a setting before starting a conversation. Each scenario card shows the location name, a brief description, and an icon or illustration.

---

## Acceptance Criteria

- [ ] Page at `/scenarios`
- [ ] 5 scenario cards rendered from `getAllScenarios()` (DT-008 data)
- [ ] Each card displays:
  - German name (large, bold)
  - English name (smaller, muted)
  - One-line description
  - Icon (emoji or SVG — e.g. 🚂 for Bahnhof, ☕ for Café)
- [ ] Clicking a card saves `scenarioId` to state/URL and navigates to `/personas?scenario={id}`
- [ ] Selected card has a visible active state (amber border)
- [ ] Hover state on each card
- [ ] Responsive grid: 1 col mobile, 2 col tablet, 3 col desktop
