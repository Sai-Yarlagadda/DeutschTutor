## Description

Convert the AI persona's text reply into spoken audio and play it in the browser. Each persona should use a distinct voice that matches their character profile.

**Recommended provider:** ElevenLabs (best German voices, per-character voice selection). Fallback: browser's `SpeechSynthesis` API with `lang="de-DE"`.

---

## Acceptance Criteria

- [ ] API route `POST /api/speak` created at `src/app/api/speak/route.ts`
- [ ] Route accepts:

```ts
{
  text: string;
  voiceId: string; // ElevenLabs voice ID mapped to persona
}
```

- [ ] Route returns audio as `audio/mpeg` stream
- [ ] `src/hooks/useAudioPlayer.ts` hook created to receive and play the audio stream
- [ ] Voice map defined in `src/lib/voices.ts`:

```ts
export const VOICE_MAP: Record<string, string> = {
  'railway-employee': 'ELEVENLABS_VOICE_ID_ANNA',
  'berlin-student':   'ELEVENLABS_VOICE_ID_LUKAS',
  'bavarian-grandma': 'ELEVENLABS_VOICE_ID_HELGA',
  'austrian-receptionist': 'ELEVENLABS_VOICE_ID_SOPHIE',
};
```

- [ ] Fallback: if ElevenLabs unavailable, use `window.speechSynthesis` with `de-DE`
- [ ] Audio plays automatically after AI reply is received
- [ ] User can mute (button in UI)

---

## Environment Variables Needed

```
ELEVENLABS_API_KEY=
```

---

## Implementation Notes

ElevenLabs streaming endpoint: `POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}/stream`

Headers: `xi-api-key: $ELEVENLABS_API_KEY`

---

## Definition of Done

AI reply text is converted to audible German speech and plays automatically in the browser. At least 2 distinct voices sound noticeably different.
