## Description

Deploy the app to Vercel and configure all environment variables. Set up a production deployment from the `main` branch.

---

## Acceptance Criteria

### Pre-Deploy Checklist

- [ ] `npm run build` passes locally with zero errors and zero TypeScript warnings
- [ ] `.env.local` exists with all required keys (never committed)
- [ ] `.env.example` committed to repo documenting all required keys:

```bash
# .env.example
ANTHROPIC_API_KEY=
ELEVENLABS_API_KEY=
OPENAI_API_KEY=       # optional — for Whisper STT fallback
```

### Vercel Setup

- [ ] Project connected to GitHub repo on Vercel dashboard
- [ ] All env vars added in Vercel project settings (not hardcoded)
- [ ] Framework preset: **Next.js** (auto-detected)
- [ ] Build command: `npm run build` (default)
- [ ] Output directory: `.next` (default)
- [ ] Node.js version: 20.x

### SQLite Note

- [ ] SQLite (`data/deutschtutor.db`) **cannot persist on Vercel** (ephemeral filesystem)
- [ ] Document this limitation in a `DEPLOYMENT.md` file
- [ ] For production persistence: document the Turso (SQLite edge) or Neon (PostgreSQL) upgrade path

### Post-Deploy Checks

- [ ] Production URL loads without errors
- [ ] API routes respond (test `/api/chat` with a test payload)
- [ ] Ambient audio files load from `public/audio/`

---

## Notes for Future

The SQLite database is perfect for local dev but will reset on every Vercel deployment. For production memory persistence, migrate to [Turso](https://turso.tech) (serverless SQLite) or PostgreSQL (Neon/Supabase).

---

## Definition of Done

Live Vercel URL provided. App loads and a test conversation completes end-to-end on the deployed URL.
