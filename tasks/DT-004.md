## Description

Create the top-level layout and routing skeleton that all screens slot into. No content yet — just the shell, nav, and page routes stubbed out.

## Acceptance Criteria

- [ ] `src/app/layout.tsx` sets global font, background color, and meta tags
- [ ] Routes stubbed (each just renders a placeholder `<h1>`):
  - `/` — Home / landing
  - `/scenarios` — Scenario selection
  - `/personas` — Persona selection
  - `/conversation` — Live conversation screen
- [ ] A minimal top nav with the app name and route links
- [ ] Responsive: works on mobile-width and desktop-width viewports
- [ ] No broken links in nav
