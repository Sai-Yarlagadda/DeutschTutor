## Description

The main conversation screen. This is the core UX — user presses a mic button, speaks German, sees the live transcript, gets an AI reply spoken aloud, and sees subtle corrections. Ambient audio plays in the background.

This is the most complex UI ticket. Wire all the engines together here.

---

## Acceptance Criteria

### Layout

- [ ] Page at `/conversation?scenario={id}&persona={id}`
- [ ] Persona info bar at top: name, occupation, city, scenario name
- [ ] Chat-style message history (scrollable, newest at bottom)
- [ ] Large mic button in the center bottom (holds to speak or tap to toggle)
- [ ] Live transcript preview while user is speaking
- [ ] Mute toggle for AI voice output
- [ ] "End Conversation" button that saves conversation to memory (DT-009) and returns to home

### Message Rendering

- [ ] User messages: right-aligned, lighter background
- [ ] AI messages: left-aligned, darker background, persona name label
- [ ] AI messages stream in word-by-word (not appear all at once)
- [ ] Correction cards (DT-010) appear below user messages when a mistake is detected

### State Machine

```
IDLE → LISTENING (mic pressed)
     → PROCESSING (mic released, STT running)
     → RESPONDING (Claude generating + TTS playing)
     → IDLE (response finished)
```

- [ ] Mic button visual state changes at each step
- [ ] User cannot press mic while AI is speaking

### Ambient Audio

- [ ] Loops the correct audio file for the scenario (from DT-014)
- [ ] Starts automatically on page load
- [ ] Volume at 25%
