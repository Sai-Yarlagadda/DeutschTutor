## Description

Persist and retrieve user state across conversations so the AI can remember the user's name, interests, proficiency level, mistakes, and recent conversations.

---

## Acceptance Criteria

### API Routes

- [ ] `GET /api/memory?userId=1` — returns full memory object
- [ ] `POST /api/memory/profile` — upsert user profile
- [ ] `POST /api/memory/mistakes` — append a repeated mistake
- [ ] `POST /api/memory/conversation` — save a completed conversation

### Memory object shape returned:

```ts
{
  profile: {
    name: string;
    level: string;
    interests: string[];
  };
  languageMemory: {
    repeatedMistakes: string[];
  };
  recentConversations: {
    persona: string;
    scenario: string;
    summary: string; // last 3 messages as preview
    createdAt: string;
  }[];
}
```

### Injection into Claude prompt

- [ ] `buildMemoryContext(userId)` function in `src/lib/memory.ts`
- [ ] Returns a string block appended to the system prompt:

```
User profile: Name is {name}, German level {level}, interested in {interests}.
Previous mistakes to gently watch for: {repeatedMistakes}.
Previous conversations: {recentConversationSummary}.
```

- [ ] Memory injected automatically in `POST /api/chat` (DT-006)

### Data limits

- [ ] Store max 10 recent conversations per user (oldest pruned automatically)
- [ ] Mistakes list capped at 20 items

---

## Definition of Done

Start a conversation, close the browser, reopen, start a new conversation — AI remembers the user's name and references a previous topic.
