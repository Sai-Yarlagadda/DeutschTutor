## Description

Set up a local SQLite database using `better-sqlite3` (synchronous, no async overhead) and define the schema needed to support the Memory Engine.

---

## Acceptance Criteria

- [ ] `better-sqlite3` installed and typed (`@types/better-sqlite3`)
- [ ] Database file lives at `data/deutschtutor.db` (gitignored)
- [ ] `src/lib/db.ts` exports a singleton DB connection
- [ ] Schema migrations applied via a seed script (`scripts/migrate.ts`)
- [ ] Three tables created:

### Tables

```sql
CREATE TABLE user_profiles (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL,
  level       TEXT NOT NULL DEFAULT 'A1',
  interests   TEXT,          -- JSON array stored as string
  created_at  TEXT DEFAULT (datetime('now'))
);

CREATE TABLE language_memory (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id         INTEGER NOT NULL REFERENCES user_profiles(id),
  repeated_mistakes TEXT,   -- JSON array
  updated_at      TEXT DEFAULT (datetime('now'))
);

CREATE TABLE conversation_history (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id     INTEGER NOT NULL REFERENCES user_profiles(id),
  persona     TEXT NOT NULL,
  scenario    TEXT NOT NULL,
  messages    TEXT NOT NULL, -- JSON array of {role, content}
  created_at  TEXT DEFAULT (datetime('now'))
);
```

- [ ] `scripts/migrate.ts` can be run with `npx tsx scripts/migrate.ts`
- [ ] `.gitignore` includes `data/`

---

## Implementation Notes

```ts
// src/lib/db.ts
import Database from 'better-sqlite3';
import path from 'path';

const db = new Database(path.join(process.cwd(), 'data/deutschtutor.db'));
db.pragma('journal_mode = WAL');
export default db;
```
