## Description

Define and manage the scenarios that shape each conversation. The Scenario Engine sets the vocabulary domain, injects scenario context into the system prompt, and maps each scenario to its ambient audio file and available personas.

---

## Acceptance Criteria

- [ ] Scenario data defined in `src/lib/scenarios.ts` as a typed constant
- [ ] Schema:

```ts
export type Scenario = {
  id: string;
  name: string;               // Display name
  nameDE: string;             // German name shown in UI
  description: string;
  audioFile: string;          // e.g. "railway.mp3"
  vocabulary: string[];       // Key German words for this setting
  personaIds: string[];       // Which personas are available here
  systemPromptAddition: string; // Extra context injected into Claude prompt
};
```

- [ ] Five scenarios defined:

| ID | Name | Audio |
|----|------|-------|
| `railway-station` | Railway Station | `railway.mp3` |
| `cafe` | Café | `cafe.mp3` |
| `hotel` | Hotel | `hotel.mp3` |
| `university` | University | `classroom.mp3` |
| `german-friend` | German Friend | `apartment.mp3` |

- [ ] `getScenario(id: string): Scenario` helper exported
- [ ] `getAllScenarios(): Scenario[]` helper exported
- [ ] Vocabulary list for each scenario has at least 8 words
- [ ] Unit test: all 5 scenarios load without error
