## Description

Validate the complete user journey from landing page to a full conversation and back. This is a manual QA pass plus a basic automated smoke test.

---

## Acceptance Criteria

### Manual Test Checklist (golden path)

- [ ] Load `/` — home page renders with no console errors
- [ ] Click through to `/scenarios` — all 5 cards appear
- [ ] Select "Bahnhof" — navigates to `/personas?scenario=railway-station`
- [ ] Anna card appears — click her — navigates to `/conversation?scenario=railway-station&persona=railway-employee`
- [ ] Ambient railway audio begins
- [ ] Press mic — browser asks for microphone permission (first time)
- [ ] Speak: *"Entschuldigung, wo fährt der Zug nach München?"*
- [ ] Transcript appears in UI
- [ ] Anna replies in German (streamed text visible)
- [ ] Anna's voice plays via TTS
- [ ] Deliberately say something wrong — correction card appears
- [ ] Click "End Conversation" — saves to memory, returns home
- [ ] Start a new conversation — Anna references the previous interaction

### Edge Cases

- [ ] No microphone access granted — error state shown gracefully
- [ ] API key missing — error state shown with friendly message, not a crash
- [ ] Network offline mid-conversation — error state, conversation history preserved

### Automated Smoke Test

- [ ] `npm test` runs a basic Jest test:
  - `getScenario('railway-station')` returns correct data
  - `getPersona('railway-employee')` returns Anna
  - `buildMemoryContext(userId)` returns a non-empty string

---

## Definition of Done

All manual checklist items pass. `npm test` passes. No critical errors in browser console.
